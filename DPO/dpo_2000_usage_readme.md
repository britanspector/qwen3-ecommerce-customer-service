# 电商客服 DPO 增强数据集（2000 对）

本数据集用于第二轮 DPO，对第一轮 DPO 测试中暴露的问题做 targeted enhancement。

## 规模
- 总偏好对：2000
- prompt_type：infer_category 1600，given_category 400
- 训练目标：回答准确、安抚情绪、给出可执行方案

## 类别分布
```json
{
  "错发漏发场景": 150,
  "商品参数场景": 450,
  "退换货场景": 300,
  "物流查询场景": 250,
  "订单修改场景": 300,
  "质量投诉场景": 300,
  "其他咨询场景": 40,
  "签收未收到场景": 150,
  "价格优惠场景": 60
}
```

## 重点修复问题
1. 商品参数类：不再编造“纯棉、支持快充、食品级硅胶、可洗碗机”等结论，必须引导查看详情页/包装标识/官方参数/型号规格。
2. 已发货订单修改：不能直接说可以改快递、取消快递、改地址；必须先核实物流状态，说明可能无法保证。
3. 超期退货与退款原因：超过售后期限不能直接承诺退货；退款原因必须如实填写。
4. 食品/母婴/质量投诉：先安抚，再提醒停止食用/使用或保留凭证，提供照片/视频/订单号，核实后按平台规则处理。
5. 物流和签收未收到：不能直接判定丢件或直接退款补发，先核实代收、驿站、快递柜、签收记录。
6. 错发漏发：需要订单、商品、面单、包装、照片等证据，确认后再处理。

## 字段说明
- `user_prompt`: 输入 prompt，包含 infer_category 或 given_category 两类。
- `chosen`: 推荐回答，统一格式为 `场景分类：xxx\n客服回复：xxx`。
- `rejected`: 错误或较差回答，主要覆盖乱承诺、编造参数、回复过短、误分类、无可执行方案等问题。
- `target_issue`: 该样本针对的问题。

## 推荐训练
第二轮 DPO 建议先做 beta 对比：
- beta=0.05：约束更弱，可能提升偏好但更易偏离。
- beta=0.1：平衡版本，通常先看这个。
- beta=0.3：约束更强，更稳但改动可能较小。

cd /root/firstTunning
bash DPO/run_dpo_beta_sweep.sh
